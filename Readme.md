# **Flutter-InternshipB01_week02**
This repository is specifically created to cover the internship task assigned by NEUROAPP. 

**The Task Content is**

**Week 2: Git & GitHub Fundamentals**
Task 2.1: GitHub Account Setup & Basic Commands (2 hours)
Create a GitHub account and set up your first repository named "Flutter-InternshipB01". Initialize it with a README.md file that includes your name, batch number, and
internship start date. Practice basic git commands: clone the repository to your local
machine, create a new file called "learning_log.txt", add and commit it, then push to
GitHub.
Steps:
1. Create GitHub account at github.com
2. Create new repository "Flutter-Internship-B01"
3. Initialize with README.md
4. Install Git on your local machine
5. Configure git with: git config --global user.name "Your Name"
6. Configure email: git config --global user.email "your.email@example.com"
7. Clone repository: git clone <repository-url>
8. Create learning_log.txt file
9. Add file: git add learning_log.txt
10.Commit: git commit -m "Add learning log"
11. Push: git push origin main

**Task 2.2: Branching & Merging Practice (2 hours)**
Create a new branch called "week2-practice" in your repository. Add a new markdown
file called "git_commands.md" listing 10 essential git commands with their
descriptions. Commit your changes, push the branch to GitHub, create a pull request,
and merge it into the main branch.
Steps:
1. Create branch: git checkout -b week2-practice
2. Create git_commands.md file with 10 commands
3. Add and commit: git add . && git commit -m "Add git commands reference"
4. Push branch: git push origin week1-practice
5. Go to GitHub repository
6. Click "Pull requests" > "New pull request"
7. Select week2-practice branch
8. Create and merge pull request
9. Delete branch after merge

**Task 2.3: Collaborative GitHub Workflow (3 hours)**
Fork a public repository (use: https://github.com/firstcontributions/first-contributions),
make a meaningful contribution by adding your name to the contributors list, create a
pull request, and document the entire process in your README.md with screenshots.
Steps:
1. Fork the repository to your account
2. Clone your forked repository
3. Create new branch: git checkout -b add-your-name
4. Add your name to Contributors.md
5. Commit changes with meaningful message
6. Push to your fork
7. Create pull request to original repository
8. Take screenshots of each step
9. Add screenshots to your repository in "screenshots" folder
10.Update README.md with the process and embedded images

**Task 2.4: README.md Mastery (2 hours)**
Create a comprehensive README.md file for your Flutter-Internship-B01 repository.
Include project title, description, table of contents, installation instructions, usage
examples, screenshots section, technologies used, and contact information. Use
proper markdown syntax including headers, lists, code blocks, tables, and image
embedding.
Steps:
1. Create detailed README.md structure
2. Add project badge using shields.io
3. Include table of contents with links
4. Add code blocks with syntax highlighting
5. Create a table for weekly progress
6. Embed at least 2 images/screenshots
7. Add links to external resources
8. Include license information
9. Format properly with headers and spacing
